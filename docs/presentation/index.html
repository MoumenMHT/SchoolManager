Context & Problem Statement The Challenge

Fitya School is a private educational institution (Primary & Middle school) in Ben Telha, Baraki, renowned for its academic and moral discipline.

Despite outstanding pedagogical results, administrative operations faced major inefficiencies due to the lack of central digital tracking. Daily processes relied almost entirely on analog, paper-based workflows.

Real-world Bottleneck

Archaic management procedures led to operational friction, delayed report generation, and fragmented communication between parents and the school administration.

❌ Analog Process

  • High risk of physical record damage or loss.
  • Tedious manual schedule grid planning.
  • No live updates for parent-teacher tracking.
  • Fragmented, paper-based payment tracking.

🚀 SchoolHub Solution

  • Secure central cloud database backups.
  • Intelligent automatic timetable generator.
  • Real-time portals for parents and teachers.
  • Integrated localized billing ledger contracts.

Built on top of Laravel 12, routing requests through specific controller actions that delegate complex computations to dedicated service classes, ensuring lightweight controllers.

Database Modeling & ORM

The storage system represents school entities. Structured as third-normal form tables with primary key indexes and relational foreign constraint cascades.

  • SQLite support: Light developer environment, migrating to MySQL/Postgres in production.

Deep Dive: Localized Sibling Billing Interactive Widget

The Algerian Market Reality

Standard SaaS models attach invoices to individual students. Algerian families expect:

  • Single Parent Contract: One billing invoice covering all siblings.
  • A-la-carte Fees: Sibling A can pay Tuition + Transport, Sibling B only Tuition.
  • Sibling Discounts: Global percentage deductions on sibling counts.
  • Automated Instalments: Spreading the annual sum over 10 monthly bills.

Billing Logic (Backend Cascade)

When a contract is stored, the backend computes the aggregate fee sums, subtracts the discount percentage, divides by contract months, and saves 10 monthly `Bill` entries.

Constraint-Satisfaction Algorithm

Generating conflict-free school timetables manually is a logistical bottleneck. SchoolHub solves this in `ScheduleController::generateAll()` using priorities and constraint scoring:

1

Priority Pre-sorting

Fetch demands. Place permanent/external teachers and high coefficient subjects first.

2

Consecutive Block Allocation

Allocate multi-hour block demands (e.g. Sports: 2h consecutive slots).

3

Hard Constraints Validation

Fit standard sessions checking: teacher conflicts, classroom limits, and level limits.

4

Fallback Mode (Soft constraints)

If slot not found, relax soft rules (e.g., maximum daily hours per class/teacher).

Deep Dive: Grading & Report Cards Interactive Calculator

Trimester Average Generation

Grades are never stored pre-calculated, preventing data syncing bugs. Normalization and average updates run on-demand in `GradingService`:

Normalization formula:
normalized_grade = (raw_grade / max_grade) * 20

Subject average formula:
subject_avg = (CC_norm*1 + Devoir_norm*1 + Composition_norm*2) / 4

Overall average formula:
overall_avg = Σ(subject_avg * coeff) / Σ(coeffs)

Smart Cache Invalidation

Computed averages are stored in a database table cache. Observers monitor grade records and automatically invalidate cache rows when grades change.
